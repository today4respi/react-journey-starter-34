Documentation API DARIAPP

Cette documentation décrit toutes les API disponibles dans l'application DARIAPP.

This documentation describes all APIs available in the DARIAPP application.

API Utilisateurs / Users API

1. Récupérer tous les utilisateurs / Get All Users

GET /api/users

Description: Récupère la liste complète de tous les utilisateurs enregistrés dans le système.

Retrieves the complete list of all users registered in the system.

Exemple avec Postman / Postman Example:

  • Méthode / Method: GET
  • URL: http://localhost:3000/api/users

Réponse / Response: Liste de tous les utilisateurs / List of all users

[
  {
    "user_id": 1,
    "nom": "Dupont",
    "prenom": "Jean",
    "email": "jean.dupont@example.com",
    "role": "user"
  },
  {
    "user_id": 2,
    "nom": "Martin",
    "prenom": "Marie",
    "email": "marie.martin@example.com",
    "role": "admin"
  }
]

2. Récupérer un utilisateur spécifique / Get User by ID

GET /api/users/:id

Description: Récupère les informations détaillées d'un utilisateur en particulier à partir de son identifiant unique.

Retrieves detailed information about a specific user based on their unique identifier.

Paramètres / Parameters:

  • id: identifiant de l'utilisateur / user identifier

Exemple avec Postman / Postman Example:

  • Méthode / Method: GET
  • URL: http://localhost:3000/api/users/1

Réponse / Response: Informations de l'utilisateur avec l'id 1 / User information with id 1

{
  "user_id": 1,
  "nom": "Dupont",
  "prenom": "Jean",
  "email": "jean.dupont@example.com",
  "role": "user"
}

3. Inscription d'un utilisateur / Register User

POST /api/users/register

Description: Permet de créer un nouvel utilisateur dans le système avec ses informations personnelles et ses identifiants de connexion.

Creates a new user in the system with their personal information and login credentials.

Corps de la requête / Request Body:

  "nom": "Dupont",
  "prenom": "Jean",
  "email": "jean.dupont@example.com",
  "password": "motdepasse123",
  "role": "user"
}

Exemple avec Postman / Postman Example:

  • Méthode / Method: POST
  • URL: http://localhost:3000/api/users/register
  • Body: raw (JSON)
  • Contenu / Content: Copier le JSON ci-dessus / Copy the JSON above

Réponse / Response: Confirmation de création de l'utilisateur / User creation confirmation

{
  "message": "Utilisateur créé avec succès",
  "userId": 3,
  "success": true
}

4. Connexion d'un utilisateur / Login User

POST /api/users/login

Description: Authentifie un utilisateur et renvoie ses informations.

Authenticates a user and returns their information.

Corps de la requête / Request Body:

{
  "email": "jean.dupont@example.com",
  "password": "motdepasse123"
}

Exemple avec Postman / Postman Example:

  • Méthode / Method: POST
  • URL: http://localhost:3000/api/users/login
  • Body: raw (JSON)
  • Contenu / Content: Copier le JSON ci-dessus / Copy the JSON above

Réponse / Response: Informations de l'utilisateur connecté / User connection information

{
  "user": {
    "id": 1,
    "nom": "Dupont",
    "prenom": "Jean",
    "email": "jean.dupont@example.com",
    "role": "user"
  }
}

5. Déconnexion d'un utilisateur / Logout User

POST /api/users/logout

Description: Déconnecte l'utilisateur actuellement connecté.

Logs out the currently connected user.

Exemple avec Postman / Postman Example:

  • Méthode / Method: POST
  • URL: http://localhost:3000/api/users/logout

Réponse / Response: Confirmation de déconnexion / Logout confirmation

{
  "message": "Déconnecté avec succès"
}

6. Récupérer l'utilisateur connecté / Get Current User

GET /api/users/me

Description: Récupère les informations de l'utilisateur actuellement connecté.

Retrieves the information of the currently connected user.

Exemple avec Postman / Postman Example:

  • Méthode / Method: GET
  • URL: http://localhost:3000/api/users/me

Réponse / Response: Informations de l'utilisateur connecté / User connection information

  "nom": "Dupont",
  "prenom": "Jean",
  "email": "jean.dupont@example.com",
  "role": "user"
}

7. Modifier un utilisateur / Update User

PUT /api/users/:id

Description: Modifie les informations d'un utilisateur existant.

Updates information for an existing user.

Paramètres / Parameters:

  • id: identifiant de l'utilisateur / user identifier

Corps de la requête / Request Body: (tous les champs sont optionnels):

{
  "nom": "Nouveau Nom",
  "prenom": "Nouveau Prénom",
  "email": "nouvel.email@example.com",
  "password": "nouveaumotdepasse",
  "role": "admin"
}

Exemple avec Postman / Postman Example:

  • Méthode / Method: PUT
  • URL: http://localhost:3000/api/users/1
  • Body: raw (JSON)
  • Contenu / Content: Copier le JSON ci-dessus (ou une partie des champs à modifier)

Réponse / Response: Confirmation de mise à jour / Update confirmation

{
  "message": "Utilisateur mis à jour avec succès"
}

8. Supprimer un utilisateur / Delete User

DELETE /api/users/:id

Description: Supprime un utilisateur.

Deletes a user.

Paramètres / Parameters:

  • id: identifiant de l'utilisateur / user identifier

Exemple avec Postman / Postman Example:

  • Méthode / Method: DELETE
  • URL: http://localhost:3000/api/users/1

Réponse / Response: Confirmation de suppression / Deletion confirmation

{
  "message": "Utilisateur supprimé avec succès"
}
